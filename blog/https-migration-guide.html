Complete HTTPS Migration Guide

Secure your site and boost SEO rankings

30 min read Updated Dec 28, 2024

Let's be real - HTTPS isn't optional anymore. Google made it a ranking factor back in 2014, and these days, browsers basically slap a "Not Secure" warning on any site that's still running on HTTP. Scary for visitors, right? But don't worry - migrating to HTTPS doesn't have to be a nightmare. This guide walks you through everything step by step, from picking the right SSL certificate to making sure your hard-earned rankings stay intact.

Before You Make the Move: Your Pre-Migration Checklist

Let's Do This: Your Migration Game Plan

1 Get That SSL Certificate Installed

Install your SSL certificate on your server. The good news? Most hosts nowadays offer one-click installation. Alternatively, you can use Let's Encrypt with Certbot - it's free and works like a charm.

# Example: Install Certbot on Ubuntu
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

2 Update Your Internal Links

Time to update all those internal links from HTTP to HTTPS. Pro tip: use relative URLs or protocol-relative URLs (//domain.com) whenever you can - makes life easier.

3 Fix Your Canonical Tags

Make sure your canonical tags are pointing to the HTTPS version of your pages:

<link rel="canonical" href="https://example.com/page" />

4 Set Up Those 301 Redirects

This one's important - redirect all your HTTP traffic to HTTPS. Using 301s (not 302s!) tells Google the move is permanent and preserves all that sweet SEO value you've built up.

5 Let Google Know via Search Console

Add your new HTTPS property to Google Search Console, submit your sitemap, and use the Change of Address tool so Google knows what's up.

.htaccess Configuration

Add these rules to redirect HTTP to HTTPS:

# Redirect HTTP to HTTPS
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# Redirect www to non-www (or vice versa)
RewriteCond %{HTTP_HOST} ^www\.(.*)$ [NC]
RewriteRule ^(.*)$ https://%1/$1 [L,R=301]

Fixing Mixed Content Issues

Mixed content occurs when HTTPS pages load HTTP resources, causing security warnings:

Common Issues

  • HTTP images and videos
  • HTTP scripts and stylesheets
  • HTTP iframes
  • HTTP AJAX requests
  • Third-party widgets

Solutions

  • Update all URLs to HTTPS
  • Use protocol-relative URLs (//domain.com)
  • Use Content Security Policy
  • Update third-party embeds
  • Test with browser DevTools
<!-- Use protocol-relative URLs -->
<script src="//cdn.example.com/script.js"></script>

<!-- Or use HTTPS explicitly -->
<script src="https://cdn.example.com/script.js"></script>

SEO Considerations for Migration

Preserve Rankings

Use 301 redirects, not 302. This passes link equity and tells Google the move is permanent.

Sitemap Updates

Update your sitemap.xml to include HTTPS URLs. Submit to Google Search Console.

Timing Matters

Migrate during low-traffic periods. Monitor rankings closely for several weeks after migration.
