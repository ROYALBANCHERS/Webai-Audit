What is Technical SEO?

Definition

Technical SEO refers to the process of optimizing a website's infrastructure and foundation to help search engines crawl, index, and render your site effectively. It focuses on improving the non-content elements of your website to enhance search visibility and user experience.

Crawlability

Making it easy for search engines to discover and access your pages

Indexability

Ensuring your pages can be stored in search engine databases

Performance

Optimizing speed, mobile-friendliness, and user experience

Crawling & Indexing Fundamentals

How Search Engines Work

1

Crawling

Search bots discover URLs and fetch web pages

2

Rendering

Browsers process HTML, CSS, and JavaScript to display the page

3

Indexing

Pages are analyzed and stored in the search engine's database

4

Ranking

Indexed pages are ranked based on relevance and quality

Robots.txt Configuration

The robots.txt file tells search engine crawlers which pages they can or cannot access.

User-agent: *
Allow: /

# Disallow specific directories
Disallow: /admin/
Disallow: /private/

# Sitemap location
Sitemap: https://example.com/sitemap.xml

Important: Robots.txt is a public file. Don't use it to hide sensitive information—use authentication instead.

Meta Robots Tags

Control indexing at the page level using meta tags:

<meta name="robots" content="index, follow">

Default: Index and follow links

<meta name="robots" content="noindex, nofollow">

Don't index or follow links

Site Architecture & Structure

A well-organized site structure helps both users and search engines navigate your content.

Flat Architecture

Keep important pages within 3-4 clicks from the homepage.

Best Practice: Homepage → Category → Product

Internal Linking

Create contextual links between related pages.

Best Practice: Use descriptive anchor text

Breadcrumb Navigation

Breadcrumbs show users their location within your site hierarchy:

  1. Home
  3. Products
  5. Category

   <a itemprop="item" href="https://example.com/">
    <span itemprop="name">Home</span>

  </li>
 </ol>
</nav>

URL Structure Best Practices

Good URLs

  • https://example.com/blog/seo-tips
  • https://example.com/products/blue-running-shoes
  • https://example.com/services/web-design

Bad URLs

  • https://example.com/p=1234
  • https://example.com/prod?id=456&cat=shoes
  • https://example.com/BLUE_RUNNING_SHOES_2024

URL Best Practices Checklist

  • Use lowercase letters
  • Separate words with hyphens
  • Keep URLs short and descriptive
  • Include target keywords naturally
  • Avoid special characters
  • Remove stop words when appropriate
  • Match URL to page title
  • Use HTTPS protocol

Canonicalization

Canonical tags tell search engines which version of a page is the "master" when duplicate content exists.

When to Use Canonical Tags

  • URL parameters (e.g., ?sort=price, ?page=2)
  • HTTP vs HTTPS versions
  • WWW vs non-WWW versions
  • Print-friendly versions
  • Syndicated or republished content

Canonical Tag Implementation

<link rel="canonical" href="https://example.com/original-page" />

Self-Referencing Canonicals: It's best practice to include a canonical tag pointing to itself on every page, even if there are no duplicates.

Understanding Redirects

301 - Permanent

Indicates the page has permanently moved.

Passes ~90-99% of link equity

Use for: Site migrations, URL changes

302 - Temporary

Indicates a temporary redirect.

Doesn't pass link equity

Use for: A/B tests, maintenance

Meta Refresh

Page-level redirect directive.

Not recommended for SEO

Use for: Almost never

Implementing 301 Redirects

Using .htaccess (Apache servers):

Redirect 301 /old-page.html https://example.com/new-page.html

HTTPS & SSL Security

Why HTTPS Matters for SEO

  • Google uses HTTPS as a ranking signal
  • Chrome marks non-HTTPS sites as "Not Secure"
  • HTTPS is required for many features (HTTP/2, Service Workers)
  • Builds user trust and credibility

HTTPS Migration Checklist

Obtain an SSL certificate (Let's Encrypt offers free certificates)

Install and configure the certificate on your server

Update all internal links to use HTTPS

Set up 301 redirects from HTTP to HTTPS

Update canonical tags to use HTTPS URLs

Update Google Search Console to HTTPS version

Update robots.txt and sitemap.xml URLs

Site Speed Optimization

Page speed is a direct ranking factor for both desktop and mobile searches.

Server-Side Optimization

  • Use quality hosting
  • Enable Gzip compression
  • Implement browser caching
  • Use a CDN (Content Delivery Network)
  • Optimize database queries

Frontend Optimization

  • Minify CSS and JavaScript
  • Defer non-critical JavaScript
  • Optimize images (WebP, compression)
  • Lazy load images and videos
  • Remove unused CSS

Browser Caching Configuration

<IfModule mod_expires.c>
  ExpiresActive On
  ExpiresByType image/jpg "access plus 1 year"
  ExpiresByType image/jpeg "access plus 1 year"
  ExpiresByType image/gif "access plus 1 year"
  ExpiresByType image/png "access plus 1 year"
  ExpiresByType text/css "access plus 1 month"
  ExpiresByType application/javascript "access plus 1 month"
</IfModule>

Core Web Vitals

Google's Core Web Vitals are specific metrics that measure user experience.

Largest Contentful Paint (LCP)

Loading

Measures loading performance. Time it takes for the main content to appear.

Good

< 2.5s

Needs Work

2.5s - 4s

Poor

> 4s

Fix: Optimize images, preload important resources, remove render-blocking resources

First Input Delay (FID)

Interactivity

Measures interactivity. Time from user's first interaction to browser response.

Good

< 100ms

Needs Work

100ms - 300ms

Poor

> 300ms

Fix: Minimize JavaScript execution time, break up long tasks, use web workers

Cumulative Layout Shift (CLS)

Visual Stability

Measures visual stability. How much the page layout shifts during loading.

Good

< 0.1

Needs Work

0.1 - 0.25

Poor

> 0.25

Fix: Include size attributes on images and videos, reserve space for ads, avoid inserting content above existing content

Mobile Optimization

Mobile-First Indexing

Google primarily uses the mobile version of your site for indexing and ranking.

  • Ensure mobile content equals desktop content
  • Use structured data on mobile pages
  • Verify images and videos are accessible on mobile
  • Test with Google's Mobile-Friendly Test

Responsive Design

One site adapts to all screen sizes

Recommended

Dynamic Serving

Different HTML based on user-agent

Complex setup

Separate URLs

m.example.com for mobile

Not recommended

Structured Data Implementation

Schema markup helps search engines understand your content and can lead to rich results.

Review

JSON-LD Implementation (Recommended)

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Your Article Title",
  "image": "https://example.com/image.jpg",
  "author": {
    "@type": "Person",
    "name": "Author Name"
  },
  "publisher": {
    "@type": "Organization",
    "name": "Your Company"
  },
  "datePublished": "2024-01-15"
}
</script>

Validation: Use Google's Rich Results Test and Schema Markup Validator to test your implementation.

International SEO

Optimize your site for different languages and geographic regions.

Hreflang Tags

Tell search engines about language and regional variations:

<link rel="alternate" hreflang="en" href="https://example.com/" />
<link rel="alternate" hreflang="en-GB" href="https://example.com/en-gb/" />
<link rel="alternate" hreflang="en-US" href="https://example.com/en-us/" />
<link rel="alternate" hreflang="es" href="https://example.com/es/" />
<link rel="alternate" hreflang="x-default" href="https://example.com/" />
