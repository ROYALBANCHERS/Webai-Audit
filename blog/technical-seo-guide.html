So, What Exactly is Technical SEO?

Definition

Think of technical SEO as the plumbing and electrical work of your website. It's everything happening behind the scenes that helps search engines discover, crawl, understand, and index your pages. While content SEO focuses on what users see, technical SEO is all about optimizing the infrastructure and foundation so Google can properly access and interpret your site.

Crawlability

Making it dead simple for search engines to discover and access your pages

Indexability

Ensuring your pages actually make it into search engine databases

Performance

Optimizing speed, mobile-friendliness, and overall user experience

Crawling & Indexing: The Fundamentals

How Search Engines Actually Work

1

Crawling

Google's bots (we call 'em spiders) scour the web, following links from page to page to discover content

2

Rendering

The browser processes your HTML, CSS, and JavaScript to display the page properly

3

Indexing

Google analyzes and stores your pages in their massive database

4

Ranking

When someone searches, Google serves up the most relevant and high-quality pages

Setting Up Your Robots.txt File

The robots.txt file is like a bouncer for your site—it tells search engine crawlers which pages they can and can't access. Here's how to set it up properly:

User-agent: *
Allow: /

# Disallow specific directories
Disallow: /admin/
Disallow: /private/

# Sitemap location
Sitemap: https://example.com/sitemap.xml

Important: Keep in mind that robots.txt is publicly visible. If you've got truly sensitive info, don't rely on robots.txt—use proper authentication instead.

Meta Robots Tags

Sometimes you need more control than robots.txt offers. That's where meta robots tags come in—they let you control indexing at the page level:

<meta name="robots" content="index, follow">

Default setting: Go ahead and index this page, and follow the links

<meta name="robots" content="noindex, nofollow">

Don't index this page or follow any links (useful for thank-you pages)

Site Architecture & Structure

Here's something a lot of people overlook: how your site is organized matters immensely. A well-structured site helps both users and search engines find what they're looking for without pulling their hair out.

Keep It Flat

The golden rule? Keep important pages within 3-4 clicks from your homepage. Nobody wants to click through seven levels of navigation to find what they need.

Best Practice: Homepage → Category → Product (simple and effective)

Internal Linking Matters

Don't leave pages orphaned. Create contextual links between related pages to help both users and Google discover content.

Best Practice: Use descriptive anchor text (not "click here")

Breadcrumb Navigation

Breadcrumbs are those handy little trails that show users exactly where they are in your site hierarchy. Google loves 'em too:

  1. Home
  3. Products
  5. Category
<nav aria-label="Breadcrumb">
 <ol itemscope itemtype="https://schema.org/BreadcrumbList">
  <li itemprop="itemListElement" itemscope itemtype="https://schema.org/ListItem">
   <a itemprop="item" href="https://example.com/">
    <span itemprop="name">Home</span>
   </a>
  </li>
 </ol>
</nav>

URL Structure That Actually Works

URLs Done Right

  • https://example.com/blog/seo-tips
  • https://example.com/products/blue-running-shoes
  • https://example.com/services/web-design

URLs That Make Google Cringe

  • https://example.com/p=1234
  • https://example.com/prod?id=456&cat=shoes
  • https://example.com/BLUE_RUNNING_SHOES_2024

The URL Best Practices Cheat Sheet

  • Stick to lowercase letters
  • Use hyphens to separate words
  • Keep URLs short and sweet
  • Include keywords naturally
  • Avoid special characters
  • Ditch unnecessary stop words
  • Match URL to page title
  • Always use HTTPS

Canonicalization: Avoiding Duplicate Content Issues

Duplicate content is a headache for Google. Canonical tags tell search engines which version of a page is the "official" one when you have multiple similar pages.

When Should You Use Canonical Tags?

  • URL parameters (like ?sort=price or ?page=2)
  • HTTP vs HTTPS versions of the same page
  • WWW vs non-WWW versions
  • Print-friendly versions of pages
  • Syndicated or republished content

How to Implement Canonical Tags

<link rel="canonical" href="https://example.com/original-page" />

Pro tip: It's actually best practice to include a self-referencing canonical tag on every page, even if there aren't any duplicates. Think of it as telling Google, "Yes, this is exactly the page I want you to rank."

Understanding Redirects (Without the Headache)

301 - Permanent

This tells Google the page has moved for good.

Passes about 90-99% of link equity

Use for: Site migrations, URL changes, restructuring

302 - Temporary

Hey Google, this is just for now.

Doesn't pass link equity

Use for: A/B tests, maintenance pages, promotions

Meta Refresh

A page-level redirect (not recommended).

Not great for SEO

Use for: Honestly? Almost never

Setting Up 301 Redirects

Here's how to do it in .htaccess (if you're on Apache):

Redirect 301 /old-page.html https://example.com/new-page.html

HTTPS & SSL: Security That Boosts SEO

Why HTTPS is Non-Negotiable in 2024

  • Google explicitly uses HTTPS as a ranking signal (since 2014!)
  • Chrome literally marks non-HTTPS sites as "Not Secure"—yikes
  • Modern features like HTTP/2 and Service Workers require HTTPS
  • Users trust secure sites more (wouldn't you?)

Your HTTPS Migration Checklist

Grab an SSL certificate (Let's Encrypt gives them away for free)

Install and configure the certificate on your server

Update all those internal links to use HTTPS

Set up 301 redirects from HTTP to HTTPS versions

Update your canonical tags to point to HTTPS URLs

Don't forget to update Google Search Console to the HTTPS version

Update robots.txt and sitemap.xml with HTTPS URLs

Site Speed: Because Nobody Likes Waiting

Here's a stat that might shock you: page speed is officially a ranking factor for both desktop and mobile. If your site takes forever to load, you're not just annoying visitors—you're actively hurting your rankings.

Server-Side Optimizations

  • Invest in quality hosting (it's worth it)
  • Enable Gzip compression
  • Set up browser caching
  • Use a CDN (Cloudflare is free to start)
  • Optimize those database queries

Frontend Optimizations

  • Minify your CSS and JavaScript files
  • Defer non-critical JavaScript
  • Optimize images (WebP is your friend)
  • Lazy load images and videos
  • Remove unused CSS (you'd be surprised how much there is)

Browser Caching Configuration

<IfModule mod_expires.c>
  ExpiresActive On
  ExpiresByType image/jpg "access plus 1 year"
  ExpiresByType image/jpeg "access plus 1 year"
  ExpiresByType image/gif "access plus 1 year"
  ExpiresByType image/png "access plus 1 year"
  ExpiresByType text/css "access plus 1 month"
  ExpiresByType application/javascript "access plus 1 month"
</IfModule>

Core Web Vitals: Metrics That Actually Matter

Google's Core Web Vitals are specific measurements that, well, measure actual user experience. They're not just arbitrary numbers—they reflect how real humans experience your site.

Largest Contentful Paint (LCP)

Loading

This measures how long it takes for the main content of your page to actually appear. You know that frustrating wait when nothing loads? Yeah, that's what LCP measures.

Good

< 2.5s

Needs Work

2.5s - 4s

Poor

> 4s

How to fix it: Optimize those hero images, preload important resources, and get rid of anything that blocks rendering

First Input Delay (FID)

Interactivity

Ever clicked something on a website and... nothing happened? That's what FID measures—the delay between when a user interacts and when the browser actually responds.

Good

< 100ms

Needs Work

100ms - 300ms

Poor

> 300ms

How to fix it: Cut down on JavaScript execution time, break up long tasks, and consider using web workers

Cumulative Layout Shift (CLS)

Visual Stability

You know when you're about to click a button and suddenly the page shifts and you click something else? That's exactly what CLS measures—how stable your page is while loading.

Good

< 0.1

Needs Work

0.1 - 0.25

Poor

> 0.25

How to fix it: Always include size attributes on images and videos, reserve space for ads, and don't insert content above existing content

Mobile Optimization: Can't Ignore It Anymore

Mobile-First Indexing is Here

Here's the reality: Google primarily uses the mobile version of your site for indexing and ranking. If your mobile site sucks, your rankings will too.

  • Make sure mobile content matches desktop content
  • Don't forget structured data on mobile pages
  • Ensure images and videos work on mobile
  • Test with Google's Mobile-Friendly Test tool

Separate URLs

JSON-LD Implementation (Recommended)

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Your Article Title",
  "image": "https://example.com/image.jpg",

International SEO: Going Global?

If you're targeting audiences in different countries or speaking different languages, you need to handle international SEO properly. Otherwise, you might end up with duplicate content issues or serving the wrong version to the wrong audience.

Hreflang Tags Explained

These tags tell Google which language and region each version of your page is targeting:

<link rel="alternate" hreflang="en" href="https://example.com/" />
<link rel="alternate" hreflang="en-GB" href="https://example.com/en-gb/" />
<link rel="alternate" hreflang="en-US" href="https://example.com/en-us/" />
<link rel="alternate" hreflang="es" href="https://example.com/es/" />
<link rel="alternate" hreflang="x-default" href="https://example.com/" />

URL Structure Options

  • ccTLDs (Best Choice)

    example.fr, example.de—signals clear geographic targeting

  • Subdirectories (Solid Option)

    example.com/fr/, example.com/de/—easy to manage

  • Subdomains (Works, But...)

    fr.example.com, de.example.com—more complex setup
